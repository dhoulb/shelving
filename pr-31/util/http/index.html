RequestHandler

(request: Request, ...args: A) => Response | Promise<Response>

A handler function takes a `Request` and optional extra arguments and returns a `Response` (possibly asynchronously).

OptionalRequestHandler

(request: Request, ...args: A) => Response | Promise<Response> | undefined

An optional request handler that may return `undefined` to indicate no match.

OptionalRequestHandlers

Iterable<OptionalRequestHandler<A>>

A list of optional request handlers.

parseRequestBody

(request: Request, caller: AnyCaller = parseRequestBody) => Promise<unknown>

Parse the body content of an HTTP `Request` based on its content type, or throw `RequestError` if the content could not be parsed.

parseRequestJSON

(request: Request, caller: AnyCaller = parseRequestJSON) => Promise<unknown>

Parse JSON from an HTTP `Request`, or return `undefined` when the request has no body.

parseRequestFormData

(request: Request, caller: AnyCaller = parseRequestFormData) => Promise<FormData | undefined>

Parse `FormData` from an HTTP `Request`, or return `undefined` when the request has no body.

parseResponseBody

(response: Response, caller: AnyCaller = parseResponseBody) => Promise<unknown>

Parse the body content of an HTTP `Response` based on its content type, or throw `ResponseError` if the content could not be parsed.

parseResponseJSON

(response: Response, caller: AnyCaller = parseResponseJSON) => Promise<unknown>

Parse JSON from an HTTP `Response`, or return `undefined` when the response has no body.

parseResponseFormData

(response: Response, caller: AnyCaller = parseResponseFormData) => Promise<FormData>

Parse `FormData` from an HTTP `Response`, or return `undefined` when the response has no body.

getResponse

(value: unknown) => Response

Get an HTTP `Response` for an unknown value.

getErrorResponse

(reason: unknown, debug = false) => Response

Get an HTTP `Response` for an unknown error value. Returns the correct `Response` based on the type of error thrown: - If `reason` is a `Response` instance, return it directly. - If `reason` is a string, return a 422 response with the string message, e.g. `"Invalid input"` - If `reason` is an `RequestError` instance, return a response with the error's message and code (but only if `debug` is true so we don't leak error details to the client). - If `reason` is an `Error` instance, return a 500 response with the error's message (but only if `debug` is true so we don't leak error details to the client). - Anything else returns a 500 response.

HTTP request methods that have no body.

RequestBodyMethod

"POST" | "PUT" | "PATCH" | "DELETE"

HTTP request methods that have a body.

HTTP request methods.

Check whether an HTTP Request method string is a supported request methods.

RequestOptions

Pick<
	RequestInit,
	"cache" | "credentials" | "headers" | "integrity" | "keepalive" | "mode" | "redirect" | "referrer" | "referrerPolicy" | "signal"
>

Configurable options for endpoint requests.

mergeRequestOptions

({ headers: aHeaders, signal: aSignal, ...a }: RequestOptions = {}, { headers: bHeaders, signal: bSignal, ...b }: RequestOptions = {}) => RequestOptions

Merge provider-level and call-level request options. - Scalar options from `b` override `a`. - Header dictionaries are merged so call-level headers override default headers by key. - Abort signals are merged, so either abort signal will cancel the request.

createHeadRequest

(method: RequestHeadMethod, url: PossibleURL, params: Nullish<PossibleURIParams>, options: RequestOptions = {}, caller: AnyCaller = createHeadRequest) => Request

Create a body-less `Request`. - `HEAD` and `GET` requests never send a body.

createTextRequest

(method: RequestMethod, url: PossibleURL, body: string, options: RequestOptions = {}, caller: AnyCaller = createTextRequest) => Request

Create a plain-text `Request`. - `HEAD` and `GET` requests never send a body.

createJSONRequest

(method: RequestBodyMethod, url: PossibleURL, body: unknown, options: RequestOptions = {}, caller: AnyCaller = createJSONRequest) => Request

Create a JSON `Request`. - `HEAD` and `GET` requests never send a body. - If the JSON body is a data object for `HEAD` or `GET`, it is appended as `?query` params instead.

createFormDataRequest

(method: RequestBodyMethod, url: PossibleURL, body: FormData, options: RequestOptions = {}, caller: AnyCaller = createFormDataRequest) => Request

Create a multipart form-data `Request`. - `HEAD` and `GET` requests never send a body.

createXMLRequest

(method: RequestBodyMethod, url: PossibleURL, data: Data, options: RequestOptions = {}, caller: AnyCaller = createXMLRequest) => Request

Create an XML `Request`. - `HEAD` and `GET` requests never send a body. - For `HEAD` and `GET`, the data object is appended as `?query` params instead.

createRequest

(method: RequestMethod, url: PossibleURL, payload: unknown, options: RequestOptions = {}, caller: AnyCaller = createRequest) => Request

Create a `Request` instance with a valid content type based on the body. - `undefined` or `null` are sent with no body. - `FormData` is sent with `multipart/formdata` - `string` is sent with `text/plain` header. - Anything else is sent as `application/json` - Expects a fully valid URL (any `{placeholders}` in the URL are not considered). - As per the HTTP spec, `GET` and `HEAD` requests cannot contain a body

assertRequestHeadPayload

(payload: unknown, method: RequestHeadMethod, caller: AnyCaller = assertRequestHeadPayload) => asserts payload is Nullish<Data>

Assert that the payload for a HEAD or GET method is a data object, null, or undefined.
