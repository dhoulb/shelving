PossibleDate

"now" | "today" | "tomorrow" | "yesterday" | Date | number | string

Values that can be converted to dates.

isDate

(value: unknown) => value is Date

Is a value a valid date? - Note: `Date` instances can be invalid (e.g. `new Date("blah blah").getTime()` returns `NaN`). These are detected and will always return `false`

assertDate

(value: unknown, caller: AnyCaller = assertDate) => asserts value is Date

Assert that a value is a `Date` instance.

getDate

(value: unknown) => Date | undefined

Convert an unknown value to a valid `Date` instance, or return `undefined` if it couldn't be converted. - Note: `Date` instances can be invalid (e.g. `new Date("blah blah").getTime()` returns `NaN`). These are detected and will always return `null`

getNow

() => Date

Get a date representing this exact moment.

getYesterday

() => Date

Get a date representing midnight of the previous day.

getToday

() => Date

Get a date representing midnight of the current day.

getTomorrow

() => Date

Get a date representing midnight of the next day.

getMidnight

(target?: PossibleDate, caller: AnyCaller = getMidnight) => Date

Get a Date representing exactly midnight of the specified date.

getMonday

(target?: PossibleDate, caller: AnyCaller = getMonday) => Date

Get a Date representing midnight on Monday of the specified week.

getMonthStart

(target?: PossibleDate, caller: AnyCaller = getMonthStart) => Date

Get a Date representing the first day of the specified month.

requireDate

(value: PossibleDate = "now", caller: AnyCaller = requireDate) => Date

Convert a possible date to a `Date` instance, or throw `RequiredError` if it couldn't be converted.

getTimestamp

(value?: unknown) => number | undefined

Convert an unknown value to a timestamp (milliseconds past Unix epoch), or `undefined` if it couldn't be converted.

requireTimestamp

(value?: PossibleDate) => number

Convert a possible date to a timestamp (milliseconds past Unix epoch), or throw `RequiredError` if it couldn't be converted.

getDateTimeString

(value?: unknown) => string | undefined

Convert an unknown value to a local date string like "2015-09-12T18:30:00", or `undefined` if it couldn't be converted.

requireDateTimeString

(value?: PossibleDate, caller: AnyCaller = requireDateTimeString) => string

Convert a possible `Date` instance to a local YMD string like "2015-09-12T18:30:00", or throw `RequiredError` if it couldn't be converted.

getDateString

(value?: unknown) => string | undefined

Convert an unknown value to a local date string like "2015-09-12", or `undefined` if it couldn't be converted.

requireDateString

(value?: PossibleDate, caller: AnyCaller = requireDateString) => string

Convert a possible `Date` instance to a local date string like "2015-09-12", or throw `RequiredError` if it couldn't be converted.

getTimeString

(value?: unknown) => string | undefined

Convert an unknown value to a local time string like "18:32:00", or `undefined` if it couldn't be converted.

requireTimeString

(value?: PossibleDate, caller: AnyCaller = requireTimeString) => string

Convert a possible `Date` instance to local time string like "18:32:00", or throw `RequiredError` if it couldn't be converted.

addYears

(change: number, target?: PossibleDate, caller: AnyCaller = addYears) => Date

Return a new date that increase or decreases the month based on an input date. - February 29th is a special cased and is _rounded down_ to February 28th on non-leap years.

addMonths

(change: number, target?: PossibleDate, caller: AnyCaller = addMonths) => Date

Return a new date that increase or decreases the month based on an input date. - Note that with Javascript "rollover" semantics, adding a month when we're on e.g. 31st of August would normally roll _past_ September and return 1st October. - To avoid this we clamp the date to the end of the month if rollover happens.
