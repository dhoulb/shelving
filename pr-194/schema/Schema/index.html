    Schemaclass

    implements Validator<T>
    new Schema<T>({ one = "value", many = `${one}s`, title, description, placeholder, value }: SchemaOptions)
    ParamTypeDefault
    optionsSchemaOptions-Options allowed by a Schema instance.
    .onestring-String for one of this thing, e.g. `product` or `item` or `sheep`
    .manystring-String for several of this thing, e.g. `products` or `items` or `sheep` (defaults to `one` + "s")
    .titlestring | undefined-Title of the schema, e.g. for using as the title of a corresponding field.
    .descriptionstring | undefined-Description of the schema, e.g. for using as a description in a corresponding field.
    .placeholderstring | undefined-Placeholder of the schema, e.g. for using as a placeholder in a corresponding field.
    .valueunknown-Default value for the schema if `validate()` is called with an `undefined` value.
    Return
    Schema<T>Schema is an object instance with a validate() method that converts unknown input into a known, valid type T.

    Schema is an object instance with a validate() method that converts unknown input into a known, valid type T.

    • Type T represents the type of value validate() returns.
    • validate() throws a string error message if the value was not valid.

    Examples

    class CustomSchema extends Schema<string> {
 	validate(unsafeValue: unknown): string {
 		if (typeof unsafeValue !== "string") throw "Must be string";
 		return unsafeValue;
 	}
 }
