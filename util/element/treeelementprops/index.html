Props for a tree element — must have a `tree-` prefixed type.

name

string

The primary identifier shown in menus, cards, and other listings. - Always set. For files this is the basename (e.g. `"array.ts"`); for directories it's the directory name; for documented symbols it's the declared name (e.g. `"getFirst"`). - `key` is typically the slugified version of `name`.

title

string | undefined

Optional visual-only override for `name`, set only when a confident source is available (e.g. a markdown `<h1>`, a docblock title). - Renderers should fall back to `name` when `title` is missing.

description

string | undefined

content

Elements | undefined

children

TreeElements | undefined

Children of a tree element must be other tree elements.
