sql

(strings: TemplateStringsArray, ...values: ImmutableArray<unknown>) => SQLFragment

Define an SQL fragment using Javascript template literal format.

sqlIdentifier

(name: string) => SQLFragment

Define an SQL fragment for an identifier, e.g. `"myTable"`

sqlExtract

(key: Segments) => SQLFragment

Define an SQL fragment that extracts a deeply nested value for comparison, e.g. `"a" #>> {"b","c"}` in Postgres

sqlConcat

(values: ImmutableArray<SQLFragment>, separator = ", ", before = "", after = "") => SQLFragment

Define an SQL fragment to generate a series of values with a separator, e.g. `"a" = 1 AND "b" = 2`

sqlSetters

(data: TT) => SQLFragment

Define an SQL fragment for setting a list of values, e.g. `"a" = 1, "b" = 2`

sqlUpdates

(updates: Updates<TT>) => SQLFragment

Define an SQL fragment for updates, e.g. `"a" = 1, "b" = "b" + 5`

sqlUpdate

({ action, key, value }: Update) => SQLFragment

Define an SQL fragment for a single update action. - Handles flat `set` and `sum` only (single-segment key). - Nested keys (multi-segment) and `with`/`omit` actions throw `UnimplementedError`. - Subclasses should override to support nested keys and array mutation actions.

sqlValues

(data: Data) => SQLFragment

Define an SQL fragment for `VALUES` syntax, e.g. `("a", "b") VALUES (1, 2)`

sqlClauses

(query: Query<Item>) => void

Define an SQL for the `WHERE`, `ORDER BY` and `LIMIT` clauses of an SQL query, e.g. e.g. ` WHERE x = 1 ORDER BY "name" LIMIT 0, 50`

sqlWhere

(query: Query<Item>) => void

Define an SQL fragment for a `WHERE` clause, e.g. ` WHERE x = 1 AND y <= 100`

sqlFilter

({ key, operator, value }: QueryFilter) => SQLFragment

Define an SQL fragment for a filter clause on a column.

sqlOrder

(query: Query<Item>) => void

Define an SQL fragment for an `ORDER BY` clause, e.g. ` ORDER BY "a" ASC, "b" DESC` - Nested keys (multi-segment) throw `UnimplementedError`.

sqlSort

({ key, direction }: QueryOrder) => SQLFragment

Define an SQL fragment for an individual column in an `ORDER BY`, e.g. `"a" ASC`

sqlLimit

(query: Query<Item>) => void

Define an SQL fragment for an `LIMIT` clause, e.g. ` LIMIT 50, 100`
